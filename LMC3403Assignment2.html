Step 1: Installing Git (~10 min)

Git is a distributed version control system that tracks changes to your code. Let’s install it to get started.

Installation Steps

  1. Download Git from git-scm.com.
  2. Run the installer, accepting default settings (recommended for beginners).
  3. Verify the installation in a terminal.
Pro Tip: On Windows, Git Bash provides a powerful command-line interface.

Check if Git is installed:

Terminal
git --version

Installation Checklist

Step 2: Configure Git (~10 min)

Configure Git with your identity to track your contributions.

Configure Identity
git config --global user.name "Neil Goyal"
git config --global user.email "neil@example.com"

The --global flag sets this configuration for all repositories on your machine.

Note: Use the email tied to your GitHub account for seamless integration.

Verify your settings:

Terminal
git config --list

Config Checklist

Step 3: Create a Repository (~10 min)

A repository ("repo") stores your project’s files and history.

GitHub Setup

  1. Sign up at GitHub.com.
  2. Click "New repository" from the "+" menu.
  3. Name it (e.g., "my-first-repo"), set visibility, and create.

Why GitHub?

GitHub enhances Git with collaboration tools like pull requests and issues.

Join GitHub

Step 4: Connect to GitHub (~15 min)

Link your local project to GitHub to share and back up your work.

Basic Workflow

Git Commands
git init
git add .
git commit -m "First commit"
git remote add origin https://github.com/username/my-first-repo.git
git push -u origin main

Git Flow

Git’s three stages: Working Directory → Staging Area → Repository.

Tip: Run git status to monitor your progress.

Step 5: Branching Basics (~10 min)

Branches isolate changes, keeping your main codebase stable.

Branch Commands

Terminal
git branch feature
git checkout feature
git add .
git commit -m "Add feature"
git checkout main
git merge feature

Why Branch?

Develop features or fixes safely, then merge when ready.

Branching Checklist

Step 6: Cloning Repositories (~10 min)

Clone existing repos to collaborate or explore others’ work.

Clone a Repo

Terminal
git clone https://github.com/username/repo.git

Replace the URL with any public repo’s address.

Why Clone?

Get a local copy of remote projects to contribute or learn.

Note: Use git pull later to update your clone.
