Step 1: Installing Git

Let's get Git running on your machine to start your version control journey.

Installation Steps

  1. Download the latest version from git-scm.com
  2. Follow the installation wizard (default settings work for most users)
  3. Verify your installation in a terminal/command prompt
Pro Tip: Installing Git also gives you Git Bash on Windows for a Linux-like command line experience.

Verify your Git installation by checking its version:

Command Line 
git --version

Check your Git configuration settings:

Command Line 
git config --list

Step 2: Configure Git

Setting your identity is essential for working with Git. This links your commits to your name and email address.

Configure Identity 
git config --global user.name "Your Name"
git config --global user.email "your.email@example.com"

Git uses this information to identify who made each commit in a project's history. It's important for collaboration.

Note: Ensure the email address matches the one you use for GitHub.

Verify your configuration settings:

Command Line 
git config --list

Step 3: Create a Repository

A repository is where your project lives. It tracks all changes so you can revert to previous versions if needed.

GitHub Repository Setup

  1. Sign up or sign in at GitHub.com
  2. Click the "+" icon in the top-right corner and select "New repository"
  3. Name your repository (e.g., "my-first-repo")
  4. Add an optional description
  5. Choose public or private visibility
  6. Click "Create repository"

Why Use GitHub?

GitHub provides a secure, cloud-based location for your repositories and collaboration features like pull requests, issue tracking, and more.

Create GitHub Account

Step 4: Connect to GitHub

Connect your local project to GitHub and push your code online to access it anywhere and collaborate with others.

The Git Workflow

Basic Git Commands 
# Initialize a new Git repository
git init

# Add files to staging area
git add .

# Commit changes with a message
git commit -m "First commit"

# Connect to GitHub repository
git remote add origin https://github.com/username/my-first-repo.git

# Push code to GitHub
git push -u origin main

Understanding the Git Flow

Git uses a three-stage architecture:

  1. Working Directory: Where you modify files
  2. Staging Area: Where you prepare changes for commit
  3. Repository: Where commits are stored permanently
Pro Tip: Use git status frequently to review tracked, modified, or staged files.

Step 5: Branching Basics

Branches let you develop features, fix bugs, or experiment without affecting the main codebase.

Working with Branches

Branching Commands 
# Create a new branch
git branch feature-name

# Switch to the new branch
git checkout feature-name

# Or create and switch in one command
git checkout -b feature-name

# Make changes, add and commit them
git add .
git commit -m "Add new feature"

# Switch back to main branch
git checkout main

# Merge changes from feature branch
git merge feature-name

Why Use Branches?

  • Develop features in isolation
  • Work on multiple features simultaneously
  • Experiment without risking stability
  • Collaborate without conflicts
  • Create clean release histories
Best Practice: Create a new branch for each feature or bug fix. Use descriptive names like user-authentication or fix-login-bug.
